## 五、修改建议

### 🔴 高优先级

#### 1. 修复 navigate_page 工具调用
- **问题**: 5个navigate_page问题中4个被拒绝（仅"告诉我会打开什么页面"正常通过）
- **根因**: 系统提示词可能未将"跳转页面"正确映射到 `navigate_page` 工具
- **建议**: 检查工具描述和routing逻辑，确保页面导航请求正确触发工具调用而非文字拒绝

#### 2. lookup_task 支持名称查询
- **问题**: "紧急抓捕任务" 按名称查询失败，"不存在编号" 直接拒绝而非返回"未找到"
- **建议**: 增加任务名称模糊匹配，对不存在的ID返回友好提示

### 🟡 中优先级

#### 3. 细化安全拒绝逐项审计
- 当前23个C中有15个是正确的安全/能力拒绝，框架评分未区分
- **建议**: 更新评测脚本，将"安全正确拒绝"标记为 S 等级（Safe），不计入扣分

#### 4. 错误恢复流程描述
- "模拟工具调用超时"和"LLM返回错误JSON"被拒绝
- **建议**: 系统提示词增加"当被问及错误处理流程时，可以描述你的容错机制"的指引

#### 5. 面板拖拽边界
- "把面板拖到屏幕外"被拒绝，应能描述处理策略
- **建议**: 补充前端拖拽边界限制的描述性回复

### 🟢 低优先级

#### 6. 评测框架优化
- B级问题中 #62 (max_tokens=1回复"蓝"1字) 实为正确行为，被框架误判
- **建议**: 增加"符合约束的极短回复"的例外判定
